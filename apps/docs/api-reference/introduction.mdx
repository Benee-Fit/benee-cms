# Benee-CMS API Reference

Welcome to the Benee-CMS API documentation. Our APIs enable you to integrate Benefits Blueprint functionality into your existing workflows and applications.

## Authentication

All API requests require authentication using API keys. You can generate API keys from your account dashboard.

```bash
curl -H "Authorization: Bearer YOUR_API_KEY" \
  https://api.benee.com/v1/endpoint
```

## Base URL

All API endpoints are available at:
```
https://api.benee.com/v1/
```

## Core Endpoints

### Document Processing
- `POST /documents/upload` - Upload insurance quote documents
- `GET /documents/{id}/status` - Check processing status
- `GET /documents/{id}/results` - Retrieve extracted data

### Project Management
- `GET /projects` - List all projects
- `POST /projects` - Create new project
- `GET /projects/{id}` - Get project details
- `PUT /projects/{id}` - Update project

### Comparisons
- `POST /comparisons` - Create comparison analysis
- `GET /comparisons/{id}` - Get comparison results
- `POST /comparisons/{id}/reports` - Generate reports

## Response Format

All API responses follow a consistent JSON format:

```json
{
  "success": true,
  "data": {
    // Response data
  },
  "message": "Success",
  "timestamp": "2023-06-14T12:00:00Z"
}
```

## Error Handling

API errors return appropriate HTTP status codes with detailed error messages:

```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid document format",
    "details": {}
  }
}
```

## Rate Limiting

API requests are rate limited to ensure fair usage:
- **Free tier**: 100 requests per hour
- **Pro tier**: 1,000 requests per hour  
- **Enterprise**: Custom limits

## SDKs and Libraries

We provide official SDKs for popular programming languages:
- JavaScript/Node.js
- Python
- PHP
- C#/.NET

## Webhooks

Configure webhooks to receive real-time notifications about document processing status, comparison completions, and other events.
