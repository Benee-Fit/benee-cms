# Quote Tool (Benefits Blueprint)

The Quote Tool is the core feature of Benee-CMS, designed to streamline the insurance quote analysis process through AI-powered document processing and intelligent market comparisons.

## Overview

The Quote Tool provides three main functions:

<CardGroup cols={3}>
  <Card title="Document Upload" icon="file-arrow-up">
    Upload and process insurance quote documents with AI extraction
  </Card>
  <Card title="Market Analysis" icon="chart-line">
    Compare quotes across multiple carriers and plan options
  </Card>
  <Card title="Report Generation" icon="file-text">
    Create professional reports for client presentations
  </Card>
</CardGroup>

## Getting Started

### Quick Actions

From the Quote Tool dashboard, you have access to these quick actions:

1. **Start a New Quote** - Begin by uploading insurance documents
2. **View Comparisons** - Access existing market analysis results
3. **Recent Reports** - Review previously generated reports

## Document Processing Workflow

### Step 1: Document Upload

<AccordionGroup>
  <Accordion icon="upload" title="Upload Process">
    1. Navigate to `/quote-tool/document-parser`
    2. Drag and drop PDF files or click to browse
    3. Categorize documents:
       - **New/Current**: Fresh quotes for analysis
       - **Renegotiated**: Updated terms after negotiations
       - **Alternative**: Alternative plan options
  </Accordion>
  <Accordion icon="cog" title="AI Processing">
    The system automatically:
    - Extracts text using PDF.co technology
    - Processes content with Google Gemini AI
    - Structures data into standardized format
    - Validates extracted information
  </Accordion>
</AccordionGroup>

### Step 2: Review and Validation

After processing, review the extracted data:

- **Coverage Types**: Verify all insurance products are correctly identified
- **Premium Amounts**: Confirm monthly and annual premium calculations
- **Carrier Information**: Check carrier names and policy details
- **Plan Options**: Review multiple plan variations if present

### Step 3: Market Comparison

Once validated, proceed to comparison analysis:

- **Full Market View**: Compare all uploaded carriers side-by-side
- **Single Carrier Analysis**: Deep dive into specific carrier offerings
- **Premium Analysis**: Calculate cost differences and percentage changes
- **Coverage Comparison**: Review benefit details across options

## Analysis Features

### Premium Comparison

- **Unit Rate Analysis**: Compare rates per $1,000 of coverage
- **Monthly Premium Totals**: Aggregate costs across all coverages
- **Volume Calculations**: Factor in number of lives covered
- **Cost Impact Analysis**: Percentage differences between options

### Coverage Details

- **Benefit Schedules**: Review coverage amounts and limits
- **Elimination Periods**: Compare waiting periods for disability coverage
- **Coinsurance Rates**: Analyze cost-sharing arrangements
- **Maximum Benefits**: Review coverage limits and caps

### Plan Options

- **Multiple Plans**: Handle documents with several plan variations
- **Plan-Specific Premiums**: Track costs for each option
- **Rate Guarantees**: Document rate lock periods
- **Effective Dates**: Track coverage start dates

## Best Practices

### Document Preparation

<Card title="Optimization Tips" icon="target">
  - Use clear, text-readable PDF documents (avoid scanned images)
  - Ensure all plan information is included in uploads
  - Group related documents by carrier or plan type
  - Verify document completeness before uploading
</Card>

### Analysis Workflow

1. **Upload Related Documents Together**: Process all quotes for a project simultaneously
2. **Validate Extracted Data**: Review AI-extracted information for accuracy
3. **Compare Consistently**: Use the same time periods and coverage amounts
4. **Document Assumptions**: Note any adjustments or interpretations made

### Report Optimization

- **Tailor to Audience**: Adjust detail level based on client sophistication
- **Highlight Key Differences**: Focus on most significant premium or coverage variations
- **Include Context**: Provide relevant market or industry benchmarks
- **Plan Recommendations**: Offer clear guidance based on analysis

## Troubleshooting

### Common Issues

<AccordionGroup>
  <Accordion icon="file-x" title="Processing Errors">
    **Document Not Processing**
    - Verify PDF is text-readable (not a scanned image)
    - Check file size (must be under 50MB)
    - Ensure document contains insurance quote information
    - Try re-uploading with a different filename
  </Accordion>
  <Accordion icon="exclamation-triangle" title="Extraction Issues">
    **Missing or Incorrect Data**
    - Review source document for completeness
    - Check if document format follows standard insurance layouts
    - Manually correct extracted data in the validation step
    - Contact support for unusual document formats
  </Accordion>
  <Accordion icon="chart-line-down" title="Comparison Problems">
    **Inconsistent Results**
    - Verify all documents are for the same client/group
    - Check effective dates are comparable
    - Ensure similar coverage amounts across quotes
    - Review plan options are appropriately matched
  </Accordion>
</AccordionGroup>
